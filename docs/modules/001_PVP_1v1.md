## Goal

Two players in the open world can **challenge each other to a turn-based duel**. Fight uses the same ability/item/switch rules as solo PvE, but both sides are human-controlled teams. Winner/loser resolved server-side; both return to exploring state.

Non-goals for v1: spectators, ranked ladder, loot from opponent, group battles.

---

## Player flow

1. Player A targets Player B (existing target panel) ‚Üí **Challenge to duel**.
2. Player B receives request (poll, same pattern as trade) ‚Üí Accept / Decline (timeout 30s).
3. Server validates: same zone, distance ‚â§ N, neither `flg_battling`, `flg_trading`, nor in dialog; both have ‚â•1 usable team animal.
4. Create `battles_pvp` session; set both `flg_battling = S`, `flg_busy = S`.
5. Both clients open combat UI (reuse `AnimasterCombat`) in **PvP mode** ‚Äî no wild sprite; show opponent team.
6. Turn order: all active animals from both players sorted by **speed** (same as solo).
7. On each turn, server waits for the **owner** of the active animal to submit action (30s timeout ‚Üí pass or auto-flee policy).
8. Battle ends: one side all fainted ‚Üí other wins; flee; disconnect forfeit (policy TBD).
9. Clear flags, write result to log, optional notification + minimal rewards (v1: none or tiny EXP).

## Technical design

### New tables (draft SQL)

```sql
-- Duel request (like trade_requests)
pvp_duel_requests (
  id_duel_request PK AI,
  id_user_ig_challenger,
  id_user_ig_target,
  id_zone,
  flg_status ENUM('pending','accepted','declined','expired','cancelled'),

  id_animal,
  team_slot,

  stats snapshot...,

);

  -- same shape as battles_solo_pve_moves + id_user_ig_actor
);
```

## Rules (v1)

| Rule | Value |
|------|-------|
| Challenge range | Same as trade / talk radius (config constant) |
| Same zone | Required |
| Team | Active team only; HP > 0 at start |
| Items | Same allow-list as solo PvE |
| Flee | Allowed; counts as loss for fleer |
| Disconnect | 60s grace then forfeit |
| Rewards | None (or +1 notification trophy) |
| Cooldown | 30s before same pair can duel again |

## Security

- All actions: verify session `id_user_ig` owns the acting animal.
- Reject moves if not current turn entity.
- Rate-limit challenge spam per account.
- Log all moves to `battles_pvp_moves` for dispute review.
